## i18n — Bilingual (English/Hebrew) with RTL

### Architecture

- **Config:** `lib/i18n.ts` — bundled JSON imports, no backend/lazy-loading
- **Provider:** `components/I18nProvider.tsx` — wraps app in layout, updates `lang` and `dir` attributes
- **Toggle:** `components/LanguageToggle.tsx` — switches between `en` and `he`
- **Translations:** `public/locales/{en,he}/translation.json`
- **Default language:** English (`lng: 'en'`, `fallbackLng: 'en'`)

### Translation Key Convention

Use nested, descriptive keys matching section structure:

```
section.subkey     → "about.heading", "experience.ariaLabel"
section.items[]    → "experience.jobs" (array of objects)
ui.action          → "language.toggle", "language.label"
```

### Adding New Translatable Content

1. Add key to `public/locales/en/translation.json`
2. Add matching key to `public/locales/he/translation.json`
3. Use `t('section.key')` in component

### Patterns by Use Case

**Simple text:**
```tsx
const { t } = useTranslation();
<p>{t('about.p1')}</p>
```

**Rich text with embedded links (Trans component):**
```tsx
<Trans
  i18nKey="about.p2"
  components={[<a key="link" href="..." />]}
/>
```

**Array of objects (jobs, projects):**
```tsx
const jobs = t('experience.jobs', { returnObjects: true }) as JobType[];
```

**Aria labels for accessibility:**
```tsx
<section aria-label={t('experience.ariaLabel')}>
```

### Content Separation Pattern

Translatable content (text, titles, descriptions) lives in JSON files. Non-translatable metadata (URLs, technology lists, icons) stays in component files as `meta` arrays, joined by index:

```tsx
// Component file — NOT translated
const experiencesMeta = [
  { companyUrl: 'https://...', technologies: ['Go', 'Docker'] },
];

// JSON file — translated
"jobs": [{ "dates": "2024 — Present", "title": "...", "company": "..." }]

// Joined in render
jobs.map((job, index) => { const meta = experiencesMeta[index]; ... })
```

### RTL Support

Hebrew (`he`) activates RTL via `I18nProvider`:
- `document.documentElement.dir = 'rtl'`
- Use logical CSS properties: `ps-2` (padding-start), `ms-1` (margin-start)
- RTL-specific transforms: `rtl:rotate-180`, `rtl:group-hover:-translate-x-1`

### Adding a New Language

1. Create `public/locales/{code}/translation.json` (copy `en` as template)
2. Add resource to `lib/i18n.ts`: `{code}: { translation: imported }`
3. Update `LanguageToggle` to cycle through languages
4. If RTL, add to the RTL check in `I18nProvider`
